Get Started

Three ways to get started with Performance Agent

Recommended

VS Code Extension

One-click setup — bundles the MCP server automatically. Zero configuration.

  1. Download the .vsix from Downloads
  2. Open VS Code → Extensions → … → Install from VSIX
  3. Open GitHub Copilot Chat (Cmd+Shift+I)
  4. Switch to Agent Mode
  5. Start using all developer tools!
Download Extension

npm Package

Install the MCP server globally via npm and configure manually.

npm install -g @performance-agent/mcp-server

Then create .vscode/mcp.json:

{
  "servers": {
    "performance-agent": {
      "command": "npx",
      "args": ["@performance-agent/mcp-server"]
    }
  }
}

No API keys needed. Works offline for test generation.

Build from Source

Clone and build for development or contributing.

git clone https://github.com/amanraj-ms/copilot-perf-agent.git
cd copilot-perf-agent/mcp-server
npm install && npm run build

Then create .vscode/mcp.json:

{
  "servers": {
    "performance-agent": {
      "command": "node",
      "args": ["/path/to/mcp-server/dist/index.js"]
    }
  }
}

Troubleshooting

Common issues and solutions

MCP Server not loading / Tools don't appear in Copilot Chat
  1. Verify .vscode/mcp.json exists with the correct absolute path to dist/index.js
  2. Restart VS Code: Cmd+Shift+P → Developer: Reload Window
  3. Check the Output panel: View → Output → select "MCP: performance-agent"
  4. Ensure npm run build completed successfully and dist/index.js exists
  5. Make sure you're in Agent Mode, not Ask or Edit mode
JMX/k6 generation fails
  1. Verify your Swagger file is valid JSON: cat swagger.json | jq .
  2. Ensure the paths object exists and contains endpoints
  3. Both Swagger 2.0 and OpenAPI 3.x formats are supported
  4. Check file encoding is UTF-8 (no BOM)
Azure VM provisioning fails
  1. Login to Azure CLI: az login then az account show
  2. Check subscription quota: Azure Portal → Subscriptions → Usage + quotas
  3. Try a different region or smaller VM size
  4. Review Terraform logs in .perf-output/terraform-logs/
VM metrics return empty
  1. Wait 5–10 minutes after test completes (Azure Monitor delay)
  2. Try a wider time range: time_range: "6h"
  3. Verify the VM is still running: az vm show -g <rg> -n <name>
  4. Ensure your account has "Monitoring Reader" role
"radon not installed" warning

Radon is optional but recommended for Python analysis. Install with:

pip install radon

The analyzer works without radon (uses regex fallback) but gives more accurate results with it.
